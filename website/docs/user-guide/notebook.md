# Notebook 使用指南

Notebook 是 Hermes File Manager 中的交互式教程文档，类似 Notion 页面或 GitHub Wiki。它以 Markdown 格式存储内容，支持预定义变量、团队共享和使用统计。

## 核心概念

### 什么是 Notebook

Notebook = Markdown 内容 + 预定义变量 + 团队共享

它不是代码笔记本（如 Jupyter），而是纯文本文档，适合：
- 操作手册 / 技术教程
- 团队知识库
- 新人 onboarding 文档
- 流程文档 / SOP

### Notebook vs 文件

| 特性 | Notebook | 普通文件 |
|------|----------|----------|
| 格式 | Markdown | 任意格式 |
| 版本控制 | 数据库记录 | 文件系统 |
| 变量 | 支持预定义变量 | 不支持 |
| 共享 | 一次性分享给团队 | 需要设置权限 |
| 使用统计 | 有 usage_count | 无 |

## 快速开始

### 创建笔记本

1. 进入某个 Space
2. 点击 "笔记本" tab
3. 点击 "+ 新建笔记本"
4. 填写：
   - **名称**: 如 "Git 入门教程"
   - **描述**: 简要说明用途
   - **标签**: 如 "教程, 新手入门"
   - **初始内容**: Markdown 格式的正文
   - **团队共享**: 勾选后团队成员可见

### 查看笔记本

1. 进入笔记本 tab
2. 点击任意笔记本卡片
3. 弹出模态框显示渲染后的 Markdown 内容

### 编辑笔记本

1. 打开笔记本详情
2. 点击 "✏️ 编辑" 按钮
3. 修改内容后保存

## 变量系统

### 预定义变量

创建笔记本时可以定义变量：

| 字段 | 说明 |
|------|------|
| `name` | 变量名，如 `$REPO_URL` |
| `default_value` | 默认值 |
| `description` | 用途说明 |
| `is_required` | 是否必填 |

### 使用变量

在笔记本内容中使用变量：

```markdown
# 项目部署

请访问 `$REPO_URL` 查看代码

变量说明：
- `$REPO_URL`: 仓库地址，如 https://github.com/example/repo
- `$BRANCH`: 分支名，默认 main
```

### 运行时填充

1. 打开笔记本详情
2. 填写变量输入框
3. 点击 "复制带变量的内容"
4. 变量值会被替换，结果复制到剪贴板

## 团队学习场景

### 场景 1: 新人 Onboarding

创建系列笔记本：

1. **环境搭建** - 开发环境配置指南
2. **代码规范** - 命名、提交规范
3. **部署流程** - 从开发到生产
4. **常见问题** - FAQ 汇总

设置 `is_shared=true` 让所有新成员都能看到。

### 场景 2: 知识库

团队维护共享知识库：

- 架构设计文档
- API 使用示例
- 运维手册
- 故障排查指南

通过标签分类：`["架构", "API", "运维"]`

### 场景 3: 操作手册

创建可执行的操作指南：

1. 创建 Workflow 描述步骤
2. 在 Notebook 中引用 workflow
3. 团队成员按指南操作

## 最佳实践

### 内容组织

1. **标题层次清晰**: 使用 `#`, `##`, `###`
2. **代码块标注**: 使用 ```language 说明
3. **表格整理**: 信息用表格展示
4. **列表分步**: 步骤用有序列表

### 变量命名

- 使用 `$` 前缀: `$REPO_URL`, `$ENV`
- 描述清晰: `$DATABASE_HOST` 而非 `$HOST`
- 提供默认值: 减少填写负担

### 共享策略

| 类型 | is_shared | 适用场景 |
|------|-----------|----------|
| 个人笔记 | false | 私有备忘、学习笔记 |
| 团队共享 | true | 公共知识库、操作手册 |
| 模板 | true | 可复制的通用内容 |

## 复制与复用

### 复制笔记本

点击笔记本详情的 "📋 复制" 按钮，创建个人副本。

副本默认 `is_shared=false`，可自行修改。

### 复制带变量的内容

1. 填写变量值
2. 点击 "复制带变量的内容"
3. 变量被实际值替换，方便粘贴使用

## 常见问题

**Q: Notebook 和 Workflow 有什么区别？**
A: Notebook 是静态文档（Markdown），Workflow 是可执行命令序列。两者可配合使用：Notebook 说明步骤，Workflow 执行操作。

**Q: 可以导出为 .md 文件吗？**
A: 暂不支持导出功能，内容存储在数据库中。可通过复制内容手动导出。

**Q: 如何搜索笔记本？**
A: 通过标签筛选或使用 Space 内的搜索功能。

**Q: 删除的笔记本能恢复吗？**
A: 不能，删除操作不可逆。请确认后再删除。

## API 参考

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/spaces/{id}/notebooks` | GET | 列出空间笔记本 |
| `/api/v1/spaces/{id}/notebooks` | POST | 创建笔记本 |
| `/api/v1/notebooks/{id}` | GET | 获取详情（含内容） |
| `/api/v1/notebooks/{id}` | PATCH | 更新笔记本 |
| `/api/v1/notebooks/{id}` | DELETE | 删除笔记本 |

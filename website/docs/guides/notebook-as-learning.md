# 用 Notebook 做团队知识管理

Notebook 是 Hermes File Manager 中的交互式文档系统，适合作为团队学习资料和知识库。本文介绍如何有效组织和使用 Notebook。

## 为什么用 Notebook 做团队学习

### 相比静态文档的优势

| 优势 | 说明 |
|------|------|
| **团队可见性** | 一次分享，团队成员随时访问 |
| **变量填充** | 预定义变量，运行时动态替换 |
| **使用追踪** | 知道哪些内容最受欢迎 |
| **版本管理** | 所有人看到的是最新版本 |
| **权限控制** | 可精确控制谁能看谁能改 |

### 适用场景

- 新员工 onboarding 文档
- 技术架构说明
- 操作手册 / SOP
- 故障排查指南
- API 使用示例

## 组织结构

### 按 Space 组织

推荐结构：

```
Team Space (团队空间)
├── 📚 知识库
│   ├── Git 入门教程 (Notebook)
│   ├── CI/CD 部署指南 (Notebook)
│   └── 代码审查规范 (Notebook)
├── 🔧 操作手册
│   ├── 数据库备份流程 (Notebook)
│   └── 监控告警处理 (Notebook)
└── 📖 新人 Onboarding
    ├── 第一周任务 (Notebook)
    └── 开发环境配置 (Notebook)
```

### 标签策略

创建 Notebook 时添加标签：

```javascript
// 推荐标签
["onboarding", "教程", "架构", "运维", "API", "最佳实践"]
```

## 内容模板

### 模板 1: 操作手册

```markdown
# {操作名称}

## 适用场景
什么时候需要执行这个操作

## 前置条件
- 环境要求
- 权限要求
- $REQUIRED_VAR（如果需要变量）

## 操作步骤
1. 步骤一
2. 步骤二
3. 步骤三

## 验证
如何确认操作成功

## 回滚方案
如果出错怎么办
```

### 模板 2: 新人 Onboarding

```markdown
# {阶段名称} - {周数}

## 本周目标
要完成什么

## 任务列表
- [ ] 任务 1
- [ ] 任务 2

## 学习资源
- $DOC_LINK（文档链接变量）
- $VIDEO_LINK（视频链接变量）

## 检查点
- [ ] 完成开发环境搭建
- [ ] 提交第一次代码
```

### 模板 3: 故障排查

```markdown
# {问题名称}

## 现象
发生了什么

## 可能原因
1. 原因 A
2. 原因 B

## 排查步骤
1. 检查 $METRIC_NAME（指标变量）
2. 查看 $LOG_PATH（日志路径变量）

## 解决方案
- 方案 A（针对原因 A）
- 方案 B（针对原因 B）

## 预防措施
如何避免再次发生
```

## 变量使用技巧

### 场景变量

定义常用变量方便运行时替换：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `$REPO_URL` | 仓库地址 | https://github.com/team/project |
| `$ENV` | 环境名称 | production, staging |
| `$DATABASE_URL` | 数据库连接 | postgresql://... |
| `$API_ENDPOINT` | API 地址 | https://api.example.com |

### 占位符格式

在内容中这样写：

```markdown
请访问 $REPO_URL 查看代码
环境：$ENV
数据库：$DATABASE_URL
```

运行时填写实际值，一键复制。

## 分享与复用

### 团队共享

创建时勾选 "团队共享"，团队成员都能看到。

### 个人副本

点击 "复制" 创建个人副本，修改不影响原文档。

### 模板复用

1. 创建通用模板 Notebook（设为团队共享）
2. 成员复制后填写自己内容
3. 原模板保持不变

---

## 最佳实践

### 内容更新

1. 重要变更后及时更新 Notebook
2. 在内容底部添加 "最后更新" 说明
3. 定期回顾内容准确性

### 权限管理

| 类型 | 谁能看 | 谁能改 |
|------|--------|--------|
| 个人笔记 | 仅自己 | 仅自己 |
| 团队文档 | 所有成员 | 作者和管理员 |
| 公共模板 | 所有成员 | 管理员 |

### 搜索发现

- 使用统一标签体系
- 笔记本名称清晰描述内容
- 在 Space 说明中放索引

## 案例：新人培训材料包

### 结构

```
新人 Onboarding Space
├── Week 1: 环境与工具
│   ├── 开发环境配置指南 (Notebook)
│   ├── Git 工作流说明 (Notebook)
│   └── 代码规范速查 (Notebook)
├── Week 2: 项目与产品
│   ├── 项目架构概述 (Notebook)
│   └── 产品技术栈 (Notebook)
└── Week 3: 实践任务
    ├── 第一个任务说明 (Notebook)
    └── 代码审查指南 (Notebook)
```

### 效果

- 新人入职后快速上手
- 减少重复讲解
- 标准化的学习路径
- 可衡量的学习进度

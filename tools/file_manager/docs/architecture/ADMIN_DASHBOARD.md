# Admin 信息可视化界面方案

## 领域分类

**领域**：企业级文件管理 / 云存储协作平台 (Enterprise File Management & Cloud Storage Collaboration Platform)

**子领域**：
- 多租户存储资源管理 (Multi-tenant Storage Resource Management)
- 基于角色的访问控制 (RBAC - Role-Based Access Control)
- 团队协作与空间隔离 (Team Collaboration & Space Isolation)

---

## 最佳实践方法论

### 1. 存储资源管理领域

**核心原则**：资源隔离 + 配额控制 + 审计追踪

**KPI 指标**：
- 存储利用率 (Storage Utilization Rate)
- 配额使用率 (Quota Usage Rate)
- 用户活跃度 (User Activity Rate)
- 资源分配效率 (Resource Allocation Efficiency)

### 2. 可视化设计原则

遵循 **Gartner Dashboards Best Practices**:
- 数据密度 vs 可读性平衡
- 关键指标突出 (Critical Metrics First)
- 层级下钻 (Drill-down Navigation)
- 异常高亮 (Anomaly Highlighting)

## 核心可视化组件

### 1. 存储池状态仪表盘

**推荐图表**：环形图 + 仪表盘

```
┌────────────────────────────────────────────────────────┐
│                   存储池概览                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│   Pool-A           Pool-B           Pool-C            │
│   [▓▓▓▓▓░░]        [▓▓▓░░░░░]       [▓▓▓▓▓▓▓▓▓▓]      │
│    68%              35%               92% ⚠            │
│                                                        │
│   500GB / 735GB    200GB / 571GB    1.8TB / 2TB        │
│                                                        │
│   [详情]            [详情]            [详情]            │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**数据指标**：
- 总量 (total_bytes)
- 已用 (used_bytes)
- 可用 (free_bytes)
- 使用率 (used_bytes / total_bytes)

### 2. 用户-空间关系桑基图

**推荐图表**：桑基图 / Force-Directed Graph

```
用户 ─────┬──────► 团队空间
          │
User-A ───┼───► Team-1 ───► Space-A
          │
User-B ───┼───► Team-1 ───► Space-B
          │              │
          │              └──► Private-C (仅 User-B)
          │
User-C ───┼───► Team-2 ───► Space-D
```

**交互**：
- 点击节点查看详情
- 悬停显示统计
- 筛选特定用户/团队/空间

### 3. 配额告警热力图

**推荐图表**：热力矩阵

```
        Space-A  Space-B  Space-C  Space-D

Team-3    🟡       🟢       🟢       🟢

```

## Admin 操作拦截规则

### 生命周期约束

| 操作 | 触发条件 | 拦截消息 | 引导操作 |
|------|---------|---------|---------|
| 删除存储池 | 有团队绑定 | "该存储池被 Team-A、Team-B 使用中，请先迁移或删除这些团队" | 查看占用的团队 |
| 创建团队 | 无可用存储池 | "系统暂无可用存储池，请联系超级管理员创建存储池" | 跳转创建存储池 |
| 删除空间 | 存在活跃成员 | "该空间仍有 5 名活跃成员，无法删除" | 查看/移除成员 |
| 上传文件 | 配额不足 | "空间配额已用尽（95%），无法上传新文件" | 申请扩容或清理 |
| 创建私人空间 | 非团队成员 | "只有团队成员才能申请私人空间" | 加入团队或创建团队 |

## 新手引导事件驱动流程

### 用户旅程地图

```
注册 ──► 加入团队 ──► 首次上传 ──► 协作编辑 ──► 进阶功能
 │          │            │            │            │
 ▼          ▼            ▼            ▼            ▼
引导1    引导2        引导3        引导4        引导5
```

### 具体引导节点

| 阶段 | 触发事件 | 引导内容 | 技术实现 |
|------|---------|---------|---------|
| 引导1 | 注册成功 | "欢迎！创建或加入第一个团队开始协作" | Modal + Tooltip |
| 引导2 | 加入团队 | "您已加入 Team-X，探索共享文件" | Highlight + Tour |
| 引导3 | 首次上传 | "文件上传成功！了解版本历史和协作功能" | Toast + Coach Mark |
| 引导4 | 多成员操作 | "检测到多人协作，学习实时同步编辑" | Video + Tutorial |
| 引导5 | 配额警告 | "空间即将用尽，清理或申请扩容" | Alert + Quick Action |

### 空间功能集成 - 工作流 & 笔记本

**工作流场景**：
- 文件归档审批：提交 → 审核 → 归档
- 团队入职流程：新成员加入 → 分配配额 → 引导完成

**笔记本场景**：
- 学习笔记：关联笔记本到 Space，持续协作编辑
- 会议纪要：实时协同编辑，自动保存历史版本

## 技术实现建议

### 前端组件

- **图表库**：ECharts / D3.js / AntV G2
- **UI 框架**：Tauri + Vue3 / React
- **状态管理**：Pinia / Redux Toolkit

### 数据层

- **实时数据**：WebSocket 推送配额更新
- **历史统计**：定时聚合到 analytics 表
- **缓存策略**：热点数据 5 分钟 TTL

## 领域最佳实践总结

**属于领域**：企业级存储资源管理与协作平台

**核心最佳实践**：

1. **分层隔离**：StoragePool → Space → Member 三层隔离
2. **配额预占**：写入前检查，避免超额
3. **操作审计**：所有操作记录审计日志
4. **资源可视化**：实时状态仪表板
5. **引导教育**：基于事件的新手引导
